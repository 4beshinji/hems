## 1. アーキテクチャ概要

```
┌─────────────────────────────────────────────────────────┐
│  Zigbee Devices (Sensors, Actuators)                    │
│  温湿度 / 人感 / ドア / プラグ / IR / ライト / カーテン     │
└──────────────────────┬──────────────────────────────────┘
                       │ Zigbee 3.0 (2.4GHz mesh)
                       ▼
              ┌────────────────┐
              │ SONOFF Dongle-E│  USB coordinator
              └───────┬────────┘
                      │ Serial
                      ▼
            ┌──────────────────┐
            │  Zigbee2MQTT     │  HA add-on / Docker
            └────────┬─────────┘
                     │ MQTT
                     ▼
            ┌──────────────────┐
            │  Home Assistant  │  state_changed events
            └────────┬─────────┘
                     │ WebSocket + REST
                     ▼
            ┌──────────────────┐
            │  HEMS ha-bridge  │  port 8016
            └────────┬─────────┘
                     │ MQTT → hems/home/{zone}/{domain}/{entity}/state
                     ▼
            ┌──────────────────┐
            │  HEMS Brain      │  WorldModel → RuleEngine → Tools
            │  (ReAct loop)    │  control_light / control_climate /
            │                  │  control_cover / get_home_devices
            └──────────────────┘
```

**ポイント**:
- Zigbee2MQTTはHA add-onとして稼働（またはDockerコンテナ）
- HEMSは既存のha-bridge経由でHA連携済み — Zigbeeデバイス追加にHEMSコード変更は基本不要
- 電源駆動のZigbeeデバイス（プラグ、リレー、電球）はルーターとしてメッシュネットワークを拡張
- センサーデータはWebSocket state_changed経由でWorldModelに自動反映

**PIR vs mmWave 比較**:

| 項目 | PIR (SNZB-03P) | mmWave (ZY-M100) |
|------|---------------|-----------------|
| 動いている人 | ✓ 検知 | ✓ 検知 |
| 静止している人（座り仕事） | ✗ 非検知 | ✓ 検知（呼吸レベル） |
| 応答速度 | <5秒 | 1-3秒 |
| 誤検知 | まれ | あり（扇風機、ペット） |
| 電池駆動 | ✓（3年+） | ✗（USB/AC電源） |

**HEMS連携**:
- mmWave → 主要居室に設置。ScheduleLearnerの到着/出発/起床パターン学習の精度向上
- PIR → 廊下・トイレに設置。高速トリガー用
- 既存のperceptionサービス（カメラ）との併用で冗長化

## 5. HEMS 機能拡張ロードマップ

### 5.1 既存機能で対応済み（コード変更不要）

| 機能 | 対応ツール/ルール | 備考 |
|------|-----------------|------|
| 照明ON/OFF/調光 | `control_light` | brightness 0-255, color_temp 153-500 |
| エアコン制御 | `control_climate` | IRブラスター経由でclimate.*エンティティ化 |
| カーテン制御 | `control_cover` | position 0-100 |
| デバイス一覧取得 | `get_home_devices` | 全デバイス状態スナップショット |
| 温度アラート抑制 | RuleEngine | 30分クールダウン |
| CO2アラート抑制 | RuleEngine | 10分クールダウン |
| 睡眠検知→照明OFF | Rule 1, 6 | 時間+姿勢+生体 |
| 起床→照明ON | Rule 4 | 活動量遷移検知 |
| 起床前カーテン開放 | Rule 3 | ScheduleLearner連携 |
| 到着前HVAC | Rule 2 | 季節別モード |
| 疲労時減光 | Rule 11 | brightness=80, color_temp=400 |

### 5.2 要拡張: 優先度高

#### (A) `control_switch` ツール追加

**現状**: スイッチは WorldModel で追跡されているが制御ツールがない。
**影響**: ZBMINI-L2リレーやスマートプラグのON/OFF制御。

```python
# tool_registry.py に追加
{
    "name": "control_switch",
    "description": "スイッチのON/OFF制御",
    "parameters": {
        "entity_id": {"type": "string", "required": True},
        "on": {"type": "boolean", "required": True}
    }
}
```

#### (B) ドアセンサー連動ルール

**新規ルール**: 玄関ドア開閉 → 到着/出発判定。

```
IF door.front_door OPEN → CLOSED (帰宅パターン)
  AND 在室センサー = 検知
  THEN speak("おかえりなさい") + 照明ON + エアコン復帰

IF door.front_door OPEN → CLOSED (外出パターン)
  AND 在室センサー = 非検知 (30秒後)
  THEN speak("いってらっしゃい") + 全デバイス省エネモード
```

#### (C) 電力モニタリング連動

**新規ルール**: スマートプラグの電力値変化による家電状態検知。

```
IF plug.washing_machine W値 > 10 → < 5 (完了検知)
  THEN create_task("洗濯物を干す", urgency=3) + speak("洗濯が終わりました")

IF plug.kettle W値 > 1000 → < 50 (沸騰完了)
```

## 付録: Zigbeeメッシュネットワーク設計メモ

### ルーターデバイス（電源駆動、メッシュ中継）
- スマートプラグ (TS011F) x2-5
- リレー (ZBMINI-L2) x2-3
- 電球 (Tuya RGBCCT) x2-3
- IRブラスター (ZS06) x1-2
- LEDコントローラー (Gledopto) x1

**合計ルーター: 8-14台** → 50台以上のエンドデバイスを安定サポート

### エンドデバイス（電池駆動、中継しない）
- 温湿度 (SNZB-02P) x3
- PIR (SNZB-03P) x2-3
- mmWave (ZY-M100) はUSB電源だがルーター機能なし
- ドア (SNZB-04P) x2
- 水漏れ (SNZB-05P) x2-3
- 振動 (DJT11LM) x1
- シーンスイッチ (TS0044) x1

**合計エンドデバイス: 14-18台** → ルーター比率十分

### 配置の原則
1. コーディネーター（ZBDongle-E）は家の中心付近に設置
2. ルーターデバイスを各部屋に最低1台配置
3. エンドデバイスからルーターまで直線で10m以内を目安
4. USB延長ケーブルでコーディネーターをサーバーから1-2m離す
